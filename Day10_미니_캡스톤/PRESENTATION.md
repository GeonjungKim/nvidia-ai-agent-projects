# 발표 대본 — TabeTabi (食べ旅) 🍜

> 예상 발표 시간 8~10분 + 데모 3분. 슬라이드 대신 이 문서 + README + 라이브 데모로 진행 가능.

---

## 1. 오프닝 (30초)

> "맛집 앱의 문제는 추천이 없어서가 아니라 **너무 많아서**입니다.
> 저는 이전에 타베로그 고신뢰 리뷰어 데이터를 크롤링해 4만 5천 곳 랭킹 앱을 만들었는데,
> 결과가 '리스트'라서 여행 계획으로 이어지지 않았습니다.
> 오늘은 그 데이터 위에 **에이전트 팀**을 얹어, '내 조건은 고정하고 나머지만 추천받는'
> 대화형 여행 컨시어지를 만들었습니다."

## 2. 데이터 자산 (1분)

- 직접 수집·정제한 타베로그 고신뢰 리뷰어 3명 병합 데이터: 레코드 47,703 → **유니크 식당 45,725곳**
- **베이지안 보정 랭킹**(bayes_score): 리뷰 1개짜리 5.0이 상위를 오염시키지 않음 (m=55, C≈3.26)
- 역 1,142곳 좌표(geojson 가공) + 검증된 지도 링크 유틸 → **이번 캡스톤은 검증된 자산의 재사용**

## 3. 왜 멀티에이전트인가 — Day9 판단 3문 (1분)

| 질문 | 답 |
|---|---|
| ① 병렬 분해 가능? | ✅ 계약 고정 후 "식당 조사"와 "활동·날씨 조사"는 독립 → 실제로 병렬 실행 (로그 교차 출력) |
| ② 가치 > 토큰 비용? | ✅ 실데이터 기반 실사용 서비스. 1회 생성 ≈ 1분, LLM 호출 10여 회 + 웹검색 ≤6회 |
| ③ 사람이 검수 가능? | ✅ 일정표 슬롯 단위 + @critic 판정 + 전체 도구 호출 로그 공개 |

**하나라도 No면 단일 에이전트로 갔을 것** — 멀티는 목적이 아니라 수단.

## 4. 아키텍처 (2분) — README 다이어그램 사용

핵심 문장 3개:
1. **"계약 먼저"** — 병렬로 흩어지기 전에 SHARED CONTRACT(지역·날짜·고정 슬롯·예산)를 고정한다. 고정 식당은 LLM이 아니라 **코드가 DB 조회로 확정**한다.
2. **"허용 도구 목록 = 권한 경계"** — @foodie는 DB 도구만, @scout은 웹검색만, @critic은 조회 1개만. 프롬프트 부탁이 아니라 구조적 강제.
3. **"리뷰어는 필수"** — 병렬 결과의 이음새는 @critic 게이트가 검증하고, 불합격이면 지적사항을 되먹여 1회 재병합(self-correction).

## 5. 환각 3중 방어 (1분)

1. @foodie는 도구 결과 밖 식당을 **말할 수 없다** (후보에 restaurant_id 필수)
2. 화면의 이름·평점·링크는 LLM 출력이 아니라 **restaurant_id로 DB 재조인**한 값
3. @critic가 유령 id·계약 위반 검증 + **코드 교정**(후보 밖 선택→1순위 교정, EXTERNAL 창작 무효, 누락 슬롯 결정론 백필)

> 실측: 병합 LLM이 'EXTERNAL:銀座 寿司匠'라는 가짜 식당을 낸 적이 있고, 코드 게이트가 무효 처리했다. (개발 로그 실화)

## 6. 라이브 데모 (3분) — 대본

**사전 체크**: `streamlit run ui.py` 실행 중 / 사이드바 "식당 45,725곳" 배지 / (가능하면) Tavily 키 활성

1. 입력창에 붙여넣기:
   > 8월 22일부터 23일까지 도쿄 1박 2일, 성인 2명이야. 라멘·스시·야키니쿠 좋아하고 저녁 예산은 1인 8,000엔 이하. 1일차 점심은 'らぁ麺や 嶋'로 고정해줘. 신주쿠 쪽에서 묵을 예정이고 서울에서 출발해!
2. **사이드바 포인팅**: SHARED CONTRACT가 채워짐 — 🔒 고정 항목, 장르가 일본어 표기로 정규화된 것
3. **🚀 버튼 클릭** → 상태창 로그 포인팅:
   - `[계약] 🔒 ... → DB 확정 (id ...)` — 고정을 코드가 잠갔다
   - `[@foodie]`와 `[@scout]` 로그가 **교차 출력** — 진짜 병렬
   - `[게이트] @critic` — 읽기 전용 검증
4. **결과 일정표**: 상단 `🤖 에이전트 작업 요약` 배지 → 시간대 타임라인(오전 활동→점심→카페→오후 활동→저녁) → 🔒 고정 반영 → 실명 호텔 추천·항공 시세 힌트 → 🚶 동선 도보 시간 → 타베로그/📍지도/🗺️경로 링크 클릭 → 하단 지도 · ✅ @critic 배지
5. (여유 시) "저녁 예산을 5,000엔으로 낮춰줘" 입력 → 계약 갱신 → ♻️ 다시 생성

**플랜 B (네트워크/LLM 장애)**: `발표_백업_데모출력.txt` (사전 실행해 둔 CLI 전체 출력) 열어서 설명.

## 7. 오늘의 엔지니어링 스토리 (1~2분, 시간 되면)

- **self-correction 실사례**: @foodie가 존재하지 않는 도구 `tokyo(...)`를 호출 → 에러를 관찰로 되먹이자 스스로 `search_restaurants`로 고쳐 재호출 (Day7 Lab4 그대로)
- **guard 훅 실동작**: 빌드 중 `.env` 파일을 자동 생성하려다 Day8에서 만든 `guard.mjs`에 차단당함 — 하네스가 에이전트(Claude)를 실제로 막았다
- **LLM에 맡길 것 vs 코드로 강제할 것**: 처음엔 고정 식당 처리를 @foodie에게 맡겼더니 external로 새거나 저녁에 중복 배치됨 → "계약 강제는 코드가 한다"로 재설계 → @critic 1차 통과
- **마무리 모드**: 도구 호출 한도에 걸리면 tools를 빼고 최종 JSON을 강제 (Day7 Lab2.5 "계획 단계엔 tools 금지"의 응용)

## 8. 한계 · 확장 (30초)

- 항공/숙소는 **검색 딥링크**까지 — 실예약은 파트너 API 필요 (1일 스코프의 의도된 경계)
- 기존 프로젝트의 Qwen 파인튜닝 추천문 생성(M4/FT)을 @foodie에 연결 가능
- MCP 서버는 `fastmcp run --transport http`로 원격화해도 루프 코드 무변경 (Day7 Lab5)

## 예상 Q&A

| 질문 | 답변 |
|---|---|
| 환각(없는 식당) 어떻게 막나 | 3중 방어: 권한 경계 → DB 재조인 → @critic + 코드 교정. 링크는 전부 DB의 실제 URL |
| 병렬이 진짜 되나 | asyncio.gather + AsyncOpenAI. 로그 타임라인에서 @foodie/@scout 교차 확인 가능 |
| 왜 예약까지 안 하나 | 항공/숙소 예약 API는 파트너 계약 필요. 실서비스도 검색 딥링크 핸드오프가 표준 패턴 |
| 데이터 윤리 | 수업 내 비공개 분석용(C-1), 리뷰 원문·리뷰어 내부 필드 미노출(C-2), DB 읽기 전용, 타베로그 네트워크 요청 0회 |
| 비용 | 1회 생성 LLM 호출 10여 회 (NVIDIA NIM Qwen), 웹검색 ≤6회, 소요 ≈1분 |
